# Task-manager

## プロジェクト概要

このプロジェクトは、Next.js（App Router）とSupabaseを使った認証付きタスク管理アプリです。

- ログイン／新規登録機能（JWT認証）
- タスク一覧表示、タスク作成、タスク編集、タスク削除機能（CRUD）
- レスポンシブ対応（スマホ・PC両対応）
- エラー発生時に表示する画面の実装（404 / 500 など）
- ログアウト機能
- Tailwind CSSによるデザイン
- Vercelにデプロイ可能

---

# 機能一覧

### 認証機能

- ユーザー新規登録
- ログイン / ログアウト

- タスク一覧表示

  - 詳細（任意 / 500文字以内）

# 使い方

### 1. ユーザー登録 / ログイン

アプリにアクセスし、以下のいずれかを行います。

- 新規ユーザーの場合  
  → **新規登録画面からアカウントを作成**

- 既存ユーザーの場合  
  → **ログイン画面からログイン**

ログイン後、タスク管理画面へ遷移します。

---

### 2. タスクの作成

「タスク作成」ボタンから新しいタスクを登録できます。

入力項目

- **タイトル**（必須 / 100文字以内）
- **詳細**（任意 / 500文字以内）
- **期限**（任意）

作成するとタスク一覧画面に表示されます。

---

### 3. タスクの確認

タスク一覧画面では

- タイトル
- 状態（完了 / 未完了）
- 期限
- 作成日

を確認できます。

タスクをクリックすると **詳細画面** に移動します。

---

### 4. タスクの編集

タスク詳細画面から **編集ボタン** を押すことで、  
タイトルや詳細、期限を変更できます。

---

### 5. タスクの削除

タスク詳細画面の **削除ボタン** からタスクを削除できます。

削除したタスクは一覧から消えます。

## セットアップ手順

1. リポジトリをクローン

```bash
git clone https://github.com/<your-username>/<project-name>.git
cd <project-name>
```

2. パッケージのインストール

```bash
npm install
# または
yarn install
```

3. 環境変数の設定（.env.local を作成）

```bash
cp .env.example .env.local
```

4. 開発サーバーの起動

```bash
npm run dev
# または
yarn dev
```

5. ブラウザで http://localhost:3000 にアクセスして動作確認

## 環境変数の設定

以下の環境変数を .env.local に設定してください：

```
NEXT_PUBLIC_SUPABASE_URL=<あなたのSupabaseプロジェクトURL>
NEXT_PUBLIC_SUPABASE_ANON_KEY=<あなたのSupabase匿名キー>

NEXT_PUBLIC_SUPABASE_URL：Supabase プロジェクトのURL

NEXT_PUBLIC_SUPABASE_ANON_KEY：公開用匿名キー（クライアントでの認証に使用）
```

注意: .env.local は絶対に Git にコミットしないこと。

## デプロイ

Vercel に接続すると、自動的にビルドされデプロイされます。
Vercel 上でも上記の環境変数を設定してください。
